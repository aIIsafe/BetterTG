## 1. What is this project?

A custom Telegram iOS client called **AnyGram**, forked from [BetterTG](https://github.com/levochkaa/BetterTG).

- **Language/stack:** Native Swift + SwiftUI, iOS 18+ deployment target
- **Telegram backend:** TDLibKit (Swift wrapper) + TDLibFramework (binary XCFramework)
- **Build method:** GitHub Actions (`aIIsafe/BetterTG`) → uploads unsigned IPA artifact
- **Install method:** [Sideloadly](https://sideloadly.io/) on Windows + free Apple ID
- **User:** Windows only, no Mac, no paid Apple ID

There is also a **second project** based on Swiftgram (see Section 8).

## 6. iOS 26 Compatibility Notes

The CI now builds with **macOS 26 / iOS 26 SDK**. Key issues and fixes:

| Issue | Fix |
|-------|-----|
| ScrollView auto-blur at edges (new iOS 26 scroll edge effect) | `compatibleScrollEdgeEffectHidden()` → `.scrollEdgeEffectStyle(.hard, for: .all)` on iOS 26 |
| `glassEffect` not available on older iOS | `compatibleGlassEffect()` → `glassEffect(.regular)` on iOS 26, `.ultraThinMaterial` fallback |
| `scrollEdgeEffectHidden` API name changed | Now it's `.scrollEdgeEffectStyle(.hard, ...)` NOT `.scrollEdgeEffectHidden` |

**All compatibility shims live in:** `BetterTG/Extensions/View/View+Compatibility.swift`

## 7. Common Build Errors & Fixes

### `'Date' is ambiguous for type lookup`
**Cause:** `TDLibKit` defines its own `Date` type conflicting with `Foundation.Date`.
**Fix:** Add `import Foundation` and use `Foundation.Date(timeIntervalSince1970: Double(int))`.

### `glassEffect is only available in iOS 26.0 or newer`
**Fix:** Wrap in `if #available(iOS 26.0, *) { ... } else { ... }` inside `@ViewBuilder` func.

### `scrollEdgeEffectHidden` compile error
**Fix:** The API is `.scrollEdgeEffectStyle(.hard, for: .all)` on iOS 26 (NOT `.scrollEdgeEffectHidden`).

### Build fails with `AVAudioSession.allowBluetoothHFP`
**Fix:** Use `.allowBluetooth` instead.

### `ScrollPosition` not available
**Fix:** Remove or guard with `#available(iOS 17, *)`.

## 8. Second Project: Swiftgram-based AnyGram

There is a SECOND build pipeline at **https://github.com/aIIsafe/anygram** that builds
AnyGram on top of Swiftgram (= full official Telegram iOS fork). This gives:
- Mini apps / bots with web apps
- Contacts, Calls, Settings tabs
- All official Telegram features

### Repo structure
```
aIIsafe/anygram/
├── .github/workflows/build.yml    ← triggers Swiftgram build
├── patches/apply_branding.py      ← changes name, bundle ID, API keys
├── patches/apply_proxy.py         ← injects proxy into AppDelegate.swift
├── patches/apply_icon.sh          ← replaces all icon sizes
└── assets/AppIcon.png             ← AnyGram 1024×1024 icon
```

Local copy: `C:\Users\aka pro\Desktop\anygram-swiftgram-build\`

### How to trigger Swiftgram build
```powershell
gh workflow run build.yml --repo aIIsafe/anygram
```

### Build time warning
⚠️ Swiftgram uses Bazel and takes **1-3 hours** to build on GitHub Actions `macos-13`.
Free tier = 2000 min/month for macOS. Use sparingly.

### Proxy injection (Swiftgram)
`patches/apply_proxy.py` creates `AnyGramProxySetup.swift` and patches `AppDelegate.swift`
to call `updateProxySettingsInteractively` with the hardcoded proxy on first launch.

## 10. Current UI Design Decisions

### Chat list
- Background: `Color.appDark` (near-black, `0.11/0.11/0.12`)
- Row height: `72pt`, transparent background
- Avatar: `56×56pt`
- Title font: `17pt semibold white`
- Preview font: `15pt gray`
- Time font: `13pt gray`
- Separator: `Rectangle(height: 0.33, color: white@0.25)` with `.padding(.leading, 84)`
- Unread badge: blue capsule

### Chat view
- Background: `Color.appDark` (NO animated gradient)
- Toolbar: `Color(r:0.10, g:0.14, b:0.26, a:0.95)` dark blue/purple
- Incoming bubble: `bubbleIncoming` (dark blue)
- Outgoing bubble: `bubbleOutgoing` (bright blue)
- Bubble corner radius: `18pt`

### Bottom tab bar (Liquid Glass)
- Shape: `Capsule` with `.ultraThinMaterial` + gradient stroke border
- Active tab: gradient blue circle indicator
- Inactive: icons at 55% opacity
- Unread badge: blue capsule on inactive tabs
- Animation: `.spring(duration: 0.35, bounce: 0.25)`
- Note: Tab bar shows Telegram FOLDER tabs (All, Work, etc.), NOT standard nav tabs.
  If user has only one folder, only "All" shows.

## 12. How to Work on This Project

### Typical workflow
1. Read this file + `AGENTS.md`
2. Make Swift changes in `BetterTG/` folder
3. Check lint errors: use `ReadLints` tool
4. Commit + push: `git add . && git commit -m "..." && git push origin main`
5. Monitor build: `gh run watch <ID> --repo aIIsafe/BetterTG`
6. Download IPA when done

### Important constraints
- Do NOT use `.scrollEdgeEffectHidden` — it's `.scrollEdgeEffectStyle(.hard, for: .all)` on iOS 26
- Do NOT use `Foundation.Date` conflicts — always `import Foundation` explicitly
- `IPHONEOS_DEPLOYMENT_TARGET = 18.0` — do not lower
- CI runs on `macos-latest` which is now **macOS 26** with iOS 26 SDK
